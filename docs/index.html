How It Works

1

You enter drug names

Type any drug names — brand or generic. "Advil" and "ibuprofen" both work. Separate multiple drugs with commas.

2

Brand names are normalized

The NIH RxNorm API automatically converts brand names to their generic equivalents before searching.

3

FDA labels are retrieved in real time

Official drug labels are fetched from the FDA OpenFDA API and stored in a local FAISS vector index for semantic search.

4

Completeness is verified

Before generating any output, the system checks that a label was retrieved for every drug. Missing labels trigger a visible warning — never a silent partial answer.

5

Plain-English summary with citations

LLaMA 3.3 70B via Groq synthesizes the retrieved label text into a caregiver-friendly summary. Every claim links back to the exact FDA label it came from.

Key Features

🔍

Source-Grounded Answers

Every claim traces back to a specific FDA DailyMed label. The model reads the document — it doesn't guess from memory.

🛡️

Silent Failure Detection

If any drug label can't be retrieved, the system blocks output and warns you — never generating a confidently incomplete answer.

💬

Plain English Output

Summaries are written for non-medical readers. No jargon. Clear severity levels and actionable next steps.

🔄

Brand Name Support

Automatically converts brand names like Advil or Tylenol to their generic equivalents using the NIH RxNorm API.

📚

Full Citations

Every output includes the FDA label ID, last updated date, and a direct link to the source document.

🔒

Privacy by Design

No personal health data collected. Users enter drug names only. The system is stateless — nothing stored between sessions.
